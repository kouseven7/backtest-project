### 追加調査 3: DSSMS戦略選択モード確認 ✅ 完了
**日時**: 2026-01-12

**問題**: なぜDSSMSではGC戦略のみ選択されたのか（main_new.pyではGC+VWAPBreakoutの2戦略）

**調査ツール**: `grep_search`（dssms_integrated_main.py確認）

**調査結果**:

#### 1. DSSMS vs main_new.py の戦略選択モード違い

**main_new.py**:
```python
selection_mode: StrategySelectionMode = StrategySelectionMode.MARKET_ADAPTIVE (デフォルト)
```
- **MARKET_ADAPTIVE**: 市場適応型
- **strong_uptrend時**: トップ2戦略を選択
- **実際の選択**: GCStrategy (0.4374) + VWAPBreakoutStrategy (0.4181)

**dssms_integrated_main.py** Line 184:
```python
self.strategy_selector = DynamicStrategySelector(
    selection_mode=StrategySelectionMode.SINGLE_BEST,  # Phase 3-C: 単一戦略選択
    ...
)
```
- **SINGLE_BEST**: 最高スコア戦略のみ
- **強制的に1戦略のみ選択**
- **実際の選択**: GCStrategy (0.4374) のみ

#### 2. selection_modeによる動作の違い

**_select_strategies_by_regime() ロジック** (dynamic_strategy_selector.py Lines 352-370):

```python
if self.selection_mode == StrategySelectionMode.MARKET_ADAPTIVE:
    # 市場レジームに応じて選択数を調整
    if 'strong' in market_regime.lower():
        # 強いトレンド: トップ2戦略
        selected = [s[0] for s in sorted_strategies[:2] if s[1] >= threshold]
    elif 'sideways' in market_regime.lower() or 'volatile' in market_regime.lower():
        # レンジ・高ボラ: トップ3戦略（分散）
        selected = [s[0] for s in sorted_strategies[:3] if s[1] >= threshold]
    else:
        # 通常トレンド: トップ2-3戦略
        selected = [s[0] for s in sorted_strategies[:2] if s[1] >= threshold]

elif self.selection_mode == StrategySelectionMode.SINGLE_BEST:
    selected = [sorted_strategies[0][0]] if sorted_strategies[0][1] >= threshold else []

elif self.selection_mode == StrategySelectionMode.TOP_N:
    selected = [s[0] for s in sorted_strategies[:3] if s[1] >= threshold]
```

**結論**: DSSMSは`SINGLE_BEST`モードで動作しているため、必然的に最高スコア1戦略のみ選択される

#### 3. 最終結論

**原因特定**: ✅ 完全に特定完了

GC戦略のみ選択される理由:
1. **selection_mode = SINGLE_BEST** (dssms_integrated_main.py Line 184)
   - 設計上、1戦略のみ選択する仕様
   - VWAPBreakoutStrategy (0.4181) はスコア2位だが選択されない

**設計意図**:
- DSSMSは「最適1銘柄 + 最適1戦略」の集中運用
- 分散投資ではなく、最も有望な1つに資金を集中させる設計
