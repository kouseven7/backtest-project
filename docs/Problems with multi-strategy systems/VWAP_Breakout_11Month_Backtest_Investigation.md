```python
    # 前提条件: 長期移動平均計算に必要なデータ数
```

#### 3.2 改善案1: ATRベースのダイナミックストップロス（推奨度: ★★★★★）

**コンセプト**: 銘柄ごとのボラティリティ（ATR: Average True Range）に応じて動的にストップロスを調整

**実装案**:
```python
# ATRベースのストップロス計算
atr_period = 14  # 14日ATRを使用
stock_data['ATR'] = calculate_atr(stock_data, period=atr_period)
current_atr = stock_data['ATR'].iloc[idx]
entry_price = stock_data['Open'].iloc[idx + 1]

# ストップロス = エントリー価格 - (ATR × 2.0)
stop_loss_price = entry_price - (current_atr * 2.0)
stop_loss_pct = (entry_price - stop_loss_price) / entry_price
```

**期待効果**:
- ✅ ボラティリティの高い銘柄（5713等）では広めのストップロス
- ✅ ボラティリティの低い銘柄（1662等）では狭めのストップロス
- ✅ 市場環境に応じた適応的なリスク管理

**リスク**:
- ❌ ボラティリティが極端に高い場合、ストップロス幅が広すぎる可能性
- ❌ ATR計算に14日分のデータが必要（ウォームアップ期間の拡大）

#### 3.3 改善案2: 時間ベースのストップロス強化（推奨度: ★★★★☆）

**コンセプト**: エントリー直後（1-2日）は特に厳格なストップロスを適用

**実装案**:
```python
# 保有期間に応じたストップロス調整
if holding_period_days <= 2:
    # エントリー直後は厳格（2%）
    stop_loss_pct = 0.02
elif holding_period_days <= 5:
    # 短期保有は標準（3%）
    stop_loss_pct = 0.03
else:
    # 中長期保有は緩和（4%）
    stop_loss_pct = 0.04
```

**期待効果**:
- ✅ エントリー直後の急落（最大損失-33,138円発生時）を防止
- ✅ 時間経過に伴い、トレンド確立を期待してストップロス緩和

**リスク**:
- ❌ 短期間での利益確保機会を逃す可能性
- ❌ 頻繁なストップロス発動により取引コストが増加

#### 3.4 改善案3: ボラティリティフィルターの導入（推奨度: ★★★☆☆）

```python
# エントリー前のボラティリティチェック
atr_period = 14

current_atr = stock_data['ATR'].iloc[idx]
current_price = stock_data['Close'].iloc[idx]
atr_ratio = current_atr / current_price

if atr_ratio > max_atr_threshold:
    # ボラティリティが高すぎる → エントリー拒否
    return 0
```

**期待効果**:
- ✅ 高ボラティリティ銘柄（5713等）への不適切なエントリーを防止
