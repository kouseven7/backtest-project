## 3. 調査結果（証拠付き）

### 3.1 項目1-1: 主要バックテストライブラリの標準動作

#### 調査内容
主要な3つのバックテストライブラリの標準エントリータイミングを確認

#### 証拠1: Zipline（Quantopian）
**ドキュメント**: Zipline公式ドキュメント
**標準動作**:
```python
# order_target_percent()は翌日始値で執行
def handle_data(context, data):
    # Day Nの終値確定後に実行
    order_target_percent(symbol, target_percent)
    # → Day N+1の始値で執行
```

**根拠**: 
- Ziplineは終値確定後にアルゴリズムを実行
- 注文は次の営業日の始値で執行
- これはルックアヘッドバイアスを防ぐための設計

#### 証拠2: Backtrader
**標準動作**:
```python
# デフォルトでは翌日始値で執行
def next(self):
    # Day Nのデータで判断
    if self.crossover > 0:
        self.buy()  # → Day N+1の始値で執行
```

**根拠**:
- `next()`メソッドはDay Nのデータで実行
- `buy()`/`sell()`は次のバーで執行
- デフォルトブローカーは始値執行

#### 証拠3: QuantConnect
**標準動作**:
```python
# SetHoldings()は翌営業日の始値で執行
def OnData(self, data):
    # Day Nの終値後に実行
    self.SetHoldings(symbol, weight)
    # → Day N+1の始値で執行
```

**根拠**:
- `OnData()`はDay N終値後に実行
- デフォルトでは翌営業日の始値で執行
- MarketOrderは即座に執行されるが、時刻はDay N+1

#### 結論
**3つのライブラリすべてで共通パターン**:
```
Day Nの終値確定後: シグナル生成
Day N+1の始値: エントリー執行
```

**業界標準**: パターンA（当日シグナル→翌日始値）

```
2025-01-14 終値時点:
  DSSMS評価: 6954=95点（最適）, 8604=88点
  → 6954を選択

2025-01-15 終値時点（仮に再計算したら）:
  DSSMS評価: 6954=82点（下落）, 8604=93点（急上昇、新たな最適）

パターンB（Option 3）:
  - 2025-01-14に6954でシグナル検出（スコア95点）
  - 2025-01-15始値でエントリー
  - この時点で6954はまだ比較的高スコア

パターンA（理想）:
  - 2025-01-15に6954でシグナル検出
  - 2025-01-16始値でエントリー
  - この時点で6954のスコアは82点（既に下落中）
  - ※ただし、現在の実装では実現不可能
```

#### メリット
1. **現在の実装で実現可能**
   - target_date-1がidx 98に位置
   - idx+1（target_date）は配列内（idx 99）
   - entry_price取得可能

2. より保守的な設計
   - 1日古いデータで判断
   - 急激な市場変動の影響を受けにくい可能性

#### デメリット
1. 業界標準ではない（機関投資家の特殊ケース）
2. DSSMS設計意図「target_date最適銘柄」と矛盾
3. リアルトレードとバックテストのタイミング不一致のリスク
4. 最適銘柄が1日でずれる可能性

### 3.7 項目3-3: 差異の影響予測

#### 理論的な差異

**要因1: 市場の変動性**
- 1日で銘柄の優位性が変わる可能性
- 特にボラティリティが高い期間

**要因2: DSSMSスコアの変化**
```
Day N-1評価: 銘柄A=95点、銘柄B=88点
Day N評価:   銘柄A=82点、銘柄B=93点

パターンB: 銘柄A選択（Day N-1評価）
パターンA: 銘柄A選択（Day N評価）※ただし実装不可能

実際のエントリー:
パターンB: Day N始値（銘柄Aスコア未確定）
パターンA: Day N+1始値（銘柄Aスコア=82点）
```

**要因3: シグナル生成タイミング**
- パターンB: Day N-2までのデータを使用（.shift(1)適用後）
- パターンA: Day N-1までのデータを使用（.shift(1)適用後）
- 1日分の情報差

#### 定量的な推定

**推定は困難**: 以下の理由により事前推定は不可能
1. 日経225銘柄のDSSMSスコア変動率が不明
2. 戦略ごとのシグナル感度が異なる
3. 期間・市場環境に依存

**実証実験が必須**: 実際のデータで測定する以外に正確な影響を把握する方法はない

- **対応**: より長期間での検証を推奨（3ヶ月以上）

**リスク4: パターンAの実現可能性**

- **対応**: Option 3を暫定措置として位置づけ
