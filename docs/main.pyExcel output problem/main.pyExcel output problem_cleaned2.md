## 🔍 **根本原因特定**

### **1. 強制決済処理の過剰適用**
**バックテストログから発見**:
```
INFO:output.data_extraction_enhancer:強制決済: ContrarianStrategy @ 6438.89
INFO:output.data_extraction_enhancer:強制決済: OpeningGapStrategy @ 6438.89
INFO:output.data_extraction_enhancer:強制決済: MomentumInvestingStrategy @ 6438.89
...（41件全てが6438.89で強制決済）
```

**問題**: 全取引が同一価格(6438.89)で強制決済され、実際のエグジットシグナル価格が無視されている。

### **2. リターン計算の根本的誤り**
**計算矛盾分析**:
```
初期資本: 1,000,000円
最終価値: 8,414,747円
実際のリターン: (8,414,747 - 1,000,000) / 1,000,000 = 741.47%
Excel表示: 7.41%

→ 小数点位置エラーまたは計算式根本的誤り
```

### **3. 取引履歴抽出ロジックの欠陥**
**問題箇所特定**: `output/simple_excel_exporter.py`
- **エントリー/エグジット価格**: バックテストログの実際価格を無視
- **保有期間計算**: 日付計算ロジック失敗
- **PnL計算**: 強制決済価格での誤計算

### **4. Unicode エンコーディング問題**
**エラーログ**:
```
UnicodeEncodeError: 'cp932' codec can't encode character '\U0001f50d'
```
**影響**: Excel出力処理が途中で失敗し、不完全なデータ生成
