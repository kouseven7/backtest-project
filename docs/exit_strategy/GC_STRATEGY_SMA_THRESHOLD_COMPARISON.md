# GC戦略 SMA乖離閾値比較調査（5.0% vs 3.0%）

**作成日**: 2026-01-28  
**調査目的**: AND条件フィルターのSMA乖離閾値を5.0%→3.0%に変更した効果を検証

---

## 目的

AND条件フィルター（トレンド強度 >= 67%ile **AND** SMA乖離 < 閾値）において、
SMA乖離閾値を**5.0%から3.0%に厳格化**した場合の影響を定量化する。

### 検証仮説
- **仮説1**: 閾値3.0%はエントリー品質をさらに向上させる（PF改善）
- **仮説2**: 閾値3.0%は取引数をさらに削減する（過剰フィルタリングのリスク）
- **仮説3**: Phase 1.6発見4（SMA乖離5%以上で大幅パフォーマンス低下）に基づき、3.0%は最適範囲

## 推奨閾値決定

**決定**: ✅ **5.0%を推奨**

**根拠**:

1. **main_new.py結果比較**:
   - 5.0%: PF=4.19, 総損益=+146.72円
   - 3.0%: PF=1.32, 総損益=+9.08円
   - **PF低下**: -68.5% (-2.87ポイント)
   - **損益低下**: -93.8% (-137.64円)

2. **DSSMS結果比較**:
   - エントリー数: 両者1件（同じ）
   - 銘柄切替: 6回 → 5回 (-16.7%)
   - 評価: 両者acceptable（同じ）

3. **フィルター分析（3.0%の問題点）**:
   - ADX高値（40-63）+ SMA乖離6-13%の**強トレンド期を全却下**
   - 例: idx=188-200（ADX=40-63, SMA_div=6-11%）→ 全拒否
   - Phase 1.6の「SMA乖離5%以上で大幅PF低下」は**極端なケース**であり、一律3.0%適用は過剰厳格

4. **理論的整合性**:
   - Phase 1.6発見: 0-5%範囲でPF=1.36（最良）
   - しかし、5.0%閾値で**PF=4.19**という高成績を達成
   - 3.0%への厳格化は「理論上の最適化」ではあるが、**実運用で性能悪化**

**次のアクション**: gc_strategy_signal.pyを5.0%に戻す（2026-01-27版に復元）

## Phase 1.6発見4との整合性

**Phase 1.6発見4**: 7203.T（トヨタ）でSMA乖離5%以上でPF大幅低下
- 0-5%: PF=1.36
- 5-10%: PF=0.34
- 10%+: PF=0.00

**3.0%閾値の理論的根拠**:
- Phase 1.6の結果から、SMA乖離は小さいほど良好
- 5.0%は「ギリギリ許容範囲」、3.0%は「より安全な範囲」
- 急騰後の暴落リスクをさらに削減
