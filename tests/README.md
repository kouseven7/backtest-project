# テストフォルダ構造

## フォルダ説明

### core/
継続的に使用する基本テスト
- 機能の回帰テスト
- 重要な検証ロジック
- 繰り返し実行するテスト

### integration/
統合テスト
- 複数モジュール間の連携テスト
- エンドツーエンドテスト
- システム全体の動作確認

### temp/
一時的な検証テスト（成功後自動削除）
- 新機能の動作確認
- バグ修正の検証
- 一度きりの確認作業

## temp/フォルダのルール

### 削除基準
1. すべてのアサーションが成功
2. 実データでの検証完了
3. フォールバックなしで動作確認済み
4. copilot-instructions.md準拠の検証完了

### ワークフロー
1. temp/でテスト作成・実行
2. 成功した場合:
   - 結果をログに記録
   - テストファイルを削除
3. 繰り返し使用する場合:
   - core/またはintegration/に移動

### 自動削除スクリプト
`tests/cleanup_temp_tests.py`を使用して一時テストを管理

## 注意事項
- temp/フォルダの内容はGitで管理されません（.gitignore設定済み）
- 削除前に必ずテスト結果をdocs/test_history/に記録
- 不明な点があれば削除せず保留
